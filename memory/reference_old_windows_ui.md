**"old windows software ui frontend"** refers to the aesthetic shown in the saved screenshot: a collection of classic Windows 95/98/XP era UI components.

Key visual characteristics:
- **Window chrome**: Blue title bar with white text, minimize/maximize/close buttons as small square pixel buttons (with X, □, −), grey beveled window border
- **Buttons**: Raised 3D bevel effect, grey background, black text — "OK", "Cancel", "Yes/Y", "No/N", "Fix", "Continue"
- **Form fields**: Sunken inset white input boxes with thin dark border
- **Progress bars**: Chunky segmented blue fill bar on grey trough (as seen in "Saving 98%")
- **Icons**: Flat 16–32px pixel icons — yellow folder, floppy disk, globe, computer, hourglass cursor, arrow cursor, red X error, green checkmark
- **Dialog types represented**: Login/welcome, install prompt, system error, error, saving progress, setup complete, notes/text editor, web browser, command prompt (black bg, monospace), media player, audio player, file explorer
- **Audio/Player UI**: Dark black player area, small transport controls (play, pause, stop, skip, rewind), gear icon, thin scrubber bar
- **Color palette**: Mid-grey (#c0c0c0) window backgrounds, royal blue (#0000aa–#0040aa) title bars, white content areas, yellow folder icons, primary-color system icons
- **Typography**: Small pixel/bitmap sans-serif (MS Sans Serif / Tahoma), tight spacing, no anti-aliasing feel
- **Overall feel**: Purely functional, skeuomorphic OS chrome, every element familiar from Windows desktop computing era

When the user says "old windows software ui frontend" they want UI to evoke this exact system — beveled grey panels, blue title bars, pixel icons, chunky dialog boxes, and the full Windows 9x/XP visual language.
